in0.

The terminal that tells you when your agent is done.

in0 is a macOS workspaces / tabs / splits terminal app built on libghostty. Every tab shows a live status dot driven by hooks from Claude Code, Codex, and OpenCode — so you can step away without missing the moment a turn finishes (or asks for input).

v0.1.1 macOS 14+ · Apple Silicon · ~14 MB DMG

Live agent status

Sidebar dot shows running / idle / needs-input / finished per workspace, broadcast over a Unix socket from the agent hook scripts. Tooltip carries the last assistant message.

Workspaces · tabs · splits

Persistent layout across restarts. Drag to reorder, double click to rename, ⌘⌥arrows to focus a neighbor pane. Splits inherit pwd from the pane you branched off of.

Quick actions

Built-in launchers for claude, codex, opencode, gitui. Add custom rows in Settings — anything you'd type plus a name.

Resume mid-session

Each agent's most recent --resume command is persisted per terminal. Reopen in0, hit the Quick Action, and you're back where you left off.

Or build from source

git clone https://github.com/yourorg/in0
cd in0
./scripts/build-vendor.sh        # libghostty.a (~30 min first run)
./scripts/regen-project.sh       # generate Xcode project
xcodebuild -project in0.xcodeproj -scheme in0 build

Enable agent hooks (one-time, per shell)

# bash
[ -f "$IN0_AGENT_HOOKS_DIR/bootstrap.bash" ] && source "$IN0_AGENT_HOOKS_DIR/bootstrap.bash"

# zsh — auto-activates via ZDOTDIR shim, no rc edit needed

# fish
if set -q IN0_AGENT_HOOKS_DIR
    source "$IN0_AGENT_HOOKS_DIR/bootstrap.fish"
end
